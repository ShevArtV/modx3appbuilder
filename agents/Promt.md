# Задача
Сделать сборщик пакетов для CMS Modx Revolution 3.

# Референсы по коду
- @_build - сборщик пакета miniShop3 как образец сборщика пакета под Modx 3.
- @modxbuilder - сборщик пакета SendIt для Modx 2, использует устаревшие методы, но имеет более широкую функциональность и правильную структуру
- @core/model - модели ядра Modx 3.
- @core/src - ядро Modx 3.
- @core/components/minishop3 - пример правильной структуры бэкенд части пакета для Modx 3.
- @assets/components/minishop3 - пример правильной структуры фронтэнд части пакета для Modx 3.
- @encrypte - информация о внедрении защиты пакета для репозитория modstore.

# Требуемая функциональность

## На этапе разработке

- По xml-схеме создавать\обновлять\удалять SQL-классы объектов и создавать\обновлять\удалять таблицы в БД.
- Создавать категорию элементов в админке с именем компонента.
- Создавать\обновлять\удалять элементы в админке из php-массивов (как здесь @_build/elements), в частности: TV, чанки, сниппеты, шаблоны, системные настройки, меню, системные события, 
  политики доступа, шаблоны политик доступа, плагины. При этом чанки, сниппеты и шаблоны могут быть статичными, т.е. их код хранится в файлах, а запись в БД только 
  ссылается на него.


## При сборке
- Собирать разные пакеты в рамках одного сайта как @modxbuilder
- Настраивать сборку через файл конфигурации
- Шифровать код для платных пакетов по инструкции из @encrypte

# Вспомогательные инструменты
- Скрипт, который найдёт в коде все текстовые сообщения и вынесет их в файлы лексикона в качестве ключей указав class_method_line в нижнем регистре
- Скрипт, который найдёт в коде все вызовы $modx->getOption(), в которых первым аргументом передано значение системной настройки с префиксом пакета и по этим значения 
  сформируется файл лексикона для системных настроек.
- Скрипт, который найдёт в коде все вызовы $modx->lexicon(), в которых первым аргументом передано значение лексикона с префиксом пакета и вынесет эти значения в дефолтный 
  файл лексиконов.
