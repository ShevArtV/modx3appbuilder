# Package Builder

**Package Builder** — CLI-инструмент для быстрого старта разработки пакетов под MODX Revolution 3. Одна команда — и вы получаете готовую структуру компонента с настроенными инструментами. Дальше пишете код, а Package Builder берёт на себя рутину: добавление элементов в MODX без входа в админку, извлечение лексиконов и настроек прямо из кода, сборку transport-пакета.

## Что даёт Package Builder

### Быстрый старт
- **Готовая структура пакета** за одну команду — директории, конфиги, шаблоны файлов, всё по стандартам MODX 3
- **Кастомные шаблоны** — создайте свой набор шаблонов для разных типов пакетов и используйте повторно

### Разработка без рутины
- **Элементы без админки** — добавляйте сниппеты, чанки, плагины, настройки и меню в MODX прямо из командной строки
- **Извлечение лексиконов из кода** — сканирует PHP-код, находит все вызовы `$modx->lexicon()` и собирает ключи в файл лексикона (переводы нужно заполнить вручную)
- **Извлечение настроек из кода** — находит `$modx->getOption()`, определяет типы настроек и создаёт готовый файл для сборки
- **Генерация классов из XML-схемы** — описали структуру таблиц в XML, получили PHP-классы и таблицы в БД

### Качество кода
- **PHPStan** — статический анализатор, который находит ошибки в PHP-коде без запуска: неправильные типы аргументов, несуществующие методы, забытые проверки на null. Включён по умолчанию (level 5)
- **PHP CS Fixer** (опционально) — автоматически приводит код к стандарту PSR-12: расставляет отступы, сортирует импорты, исправляет стиль. Запустили одну команду — и весь код оформлен единообразно
- **ESLint** (опционально) — то же самое для JavaScript: находит ошибки и проблемы в JS-файлах компонента

### Сборка и дистрибуция
- **Сборка transport.zip** с поддержкой всех типов элементов MODX
- **Фильтрация файлов** при сборке через `.packignore` — технические файлы не попадут в пакет
- **Шифрование** платных пакетов через modstore.pro API

## Быстрый старт

### 1. Установка

Рекомендуем глобальную установку — команда `mxbuilder` будет доступна из любой директории:

```bash
composer global require shevartv/modx-builder
```

!!! note "PATH"
    Убедитесь, что путь к глобальным Composer-пакетам добавлен в `PATH`. Узнать путь:
    ```bash
    composer global config bin-dir --absolute
    ```

    === "Linux"

        Добавьте в `~/.bashrc`:
        ```bash
        export PATH="$HOME/.config/composer/vendor/bin:$PATH"
        ```

    === "macOS"

        Добавьте в `~/.zshrc`:
        ```bash
        export PATH="$HOME/.composer/vendor/bin:$PATH"
        ```

    === "Windows"

        Добавьте в системную переменную `PATH`:
        ```
        %USERPROFILE%\AppData\Roaming\Composer\vendor\bin
        ```

Также можно установить локально в проект:

```bash
composer require shevartv/modx-builder --dev
```

В этом случае команда вызывается как `vendor/bin/mxbuilder`. Далее во всех примерах используется `mxbuilder` (глобальная установка).

### 2. Настройка окружения (если нет MODX)

Если у вас нет установленного MODX 3 — setup скачает ядро и настроит окружение с SQLite:

```bash
mxbuilder setup
```

Если MODX уже установлен — этот шаг не нужен, сборщик найдёт `core/config/config.inc.php` автоматически.

Подробнее: [Команда setup](commands/setup.md)

### 3. Глобальные настройки (один раз)

Настройте данные автора и параметры по умолчанию — они будут использоваться во всех проектах:

```bash
mxbuilder config
```

Подробнее: [Команда config](commands/config.md)

### 4. Инициализация проекта (опционально)

Создайте локальный конфиг `mxbuilder.json` для текущего проекта:

```bash
mxbuilder init
```

Значения по умолчанию берутся из глобального конфига. Можно переопределить любое поле — например, задать другой путь к шаблонам или включить ESLint.

!!! tip "init не обязателен"
    Если вы не вызывали `init`, первый `mxbuilder create` автоматически создаст `mxbuilder.json` из глобального конфига.

Подробнее: [Команда init](commands/init.md)

### 5. Шаблоны (опционально)

Package Builder создаёт пакеты из встроенных шаблонов. Если вы хотите использовать свои — скопируйте оригинальные в нужную папку и отредактируйте:

```bash
# Скопировать оригинальные шаблоны в папку my-templates/ текущей директории
mxbuilder templates copy ./my-templates
```

Вы можете добавить в шаблоны свои файлы — сниппеты, чанки, конфиги. Всё, что лежит в папке шаблонов, будет скопировано в структуру нового пакета при создании. Файлы с расширением `.template` обрабатываются — в них заменяются плейсхолдеры (`{{package_name}}`, `{{author_name}}` и др.).

Путь к шаблонам можно задать в глобальном конфиге (`mxbuilder config`) или в локальном (`mxbuilder init`) — тогда указывать `--template` при каждом `create` не нужно. Или указать разово:

```bash
mxbuilder create shop --template=./templates/ecommerce
```

Подробнее: [Управление шаблонами](commands/templates.md)

### 6. Конфигурация пакета

Каждый пакет имеет файл конфигурации, который создаётся автоматически при `create`:

```
package_builder/packages/<name>/config.php
```

В нём задаются версия, пути, список элементов, настройки сборки и инструменты разработки. В одном проекте может быть несколько пакетов — каждый со своим конфигом.

Подробнее: [Конфигурация](configuration.md)

### 7. Создание структуры пакета

```bash
mxbuilder create mypackage --interactive
```

Команда создаёт стандартную структуру компонента MODX 3:

```
core/components/mypackage/
├── bootstrap.php              — точка входа, регистрация namespace
├── composer.json              — зависимости и скрипты
├── phpstan.neon               — конфигурация PHPStan
├── docs/
│   ├── readme.txt             — описание пакета
│   ├── license.txt            — лицензия
│   └── changelog.txt          — история изменений
├── lexicon/
│   ├── en/default.inc.php     — английские переводы
│   └── ru/default.inc.php     — русские переводы
├── schema/
│   └── mypackage.mysql.schema.xml  — XML-схема БД
├── src/
│   ├── Mypackage.php          — главный класс компонента
│   └── Plugins/
│       └── IPlugin.php        — интерфейс плагинов
└── elements/                  — файлы элементов (если выбрано)
    ├── plugins/switch.php     — единая точка входа плагинов
    └── snippets/connector.php
```

Подробнее: [Команда create](commands/create.md)

### 8. Схема базы данных (опционально)

Если ваш компонент использует свои таблицы в БД, опишите их в XML-схеме и сгенерируйте PHP-классы:

```bash
mxbuilder schema mypackage
```

Пакеты без таблиц (например, простые сниппеты или плагины) пропускают этот шаг.

Подробнее: [Команда schema](commands/schema.md)

### 9. Работа с элементами

Опишите элементы компонента в PHP-файлах (`packages/mypackage/elements/`) и добавьте их в MODX одной командой — без входа в админку:

```bash
mxbuilder elements mypackage
```

Команда создаёт или обновляет в базе данных: сниппеты, чанки, плагины, шаблоны, TV-параметры, системные настройки, пункты меню и события.

Подробнее: [Команда elements](commands/elements.md)

### 10. Извлечение данных из кода

Package Builder умеет автоматически находить лексиконы и настройки в вашем PHP-коде:

```bash
# Найти все $modx->lexicon('key') и собрать ключи в файл лексикона
# Переводы нужно заполнить вручную
mxbuilder extract-lexicons mypackage

# Найти все $modx->getOption('key') и сгенерировать файл настроек
mxbuilder extract-settings mypackage
```

Подробнее: [extract-lexicons](commands/extract-lexicons.md) | [extract-settings](commands/extract-settings.md)

### 11. Сборка пакета

```bash
mxbuilder build mypackage
```

Команда собирает transport.zip — стандартный формат пакетов MODX, который можно установить через менеджер пакетов или загрузить на modstore.pro.

Флаг `--install` автоматически установит пакет на текущий сайт после сборки. Это удобно при разработке — собрали, установили, проверили в админке:

```bash
mxbuilder build mypackage --install
```

Подробнее: [Команда build](commands/build.md)

## Поддерживаемые элементы

| Элемент | Описание |
|---------|----------|
| Chunks | Компоненты шаблонов |
| Snippets | PHP-сниппеты |
| Plugins | Плагины с привязкой к событиям |
| Templates | Шаблоны страниц |
| TV | Дополнительные поля |
| Settings | Системные настройки |
| Menus | Пункты меню админки |
| Events | Кастомные события |
| Policies | Политики доступа |
| Policy Templates | Шаблоны политик |

## Системные требования

### Минимальные (headless-режим)

- PHP 8.1+
- Composer
- Расширение PHP: `pdo_sqlite`, `zip`, `mbstring`

Достаточно для `setup`, `create`, `build`, `schema`, `extract-*`.

### Для полного функционала

- PHP 8.1+
- Composer
- MODX Revolution 3 (установленный)
- MySQL/MariaDB
- Расширение PHP: `pdo_mysql`, `zip`, `mbstring`

Необходимо для команд `elements` и `export`.
